# MVP 功能规格与开发清单

## 1. MVP 目标

在最短开发周期内，交付一个“可认领、可命名、可学习得分、可买食物、可喂食升级”的完整网页小游戏闭环。

## 2. MVP 功能边界

## 必做

- 首页介绍与开始认领
- 选择动物
- 输入动物名字
- 动物主页展示当前状态
- 学习计时或手动完成
- 学习积分奖励 / 失败扣分
- 商店购买食物
- 背包查看与喂食
- 成长值累计与升级判定
- 学习记录与积分流水

## 可延后

- 用户登录
- 云端同步
- 排行榜
- 分享海报
- 音效与复杂动画
- 装扮系统

## 3. 页面与模块映射

| 页面 | 模块 | 关键职责 |
|---|---|---|
| 首页 | `landing` | 讲清玩法，促成认领 |
| 认领页 | `adoption` | 选动物、命名、初始化数据 |
| 动物主页 | `pet-home` | 展示状态并汇总主操作入口 |
| 学习页 | `study-session` | 计时、完成、结算积分 |
| 商店页 | `shop` | 展示商品、积分校验、购买 |
| 背包页 | `inventory` | 管理库存、选择食物、喂食 |
| 记录页 | `records` | 展示学习历史、收支、升级记录 |

## 6. 核心业务规则

## 6.1 认领

- 每个玩家默认只认领 1 只动物
- 认领时写入 `pet.type`、`pet.name`
- 初始化等级、成长值、饱腹度、背包和积分

## 6.2 学习结算

- 完成学习：增加积分并写学习记录
- 放弃学习：可选扣分并写失败记录
- 每日首次完成：追加额外积分

## 6.3 商店购买

- 校验积分是否足够
- 扣减积分
- 增加背包库存
- 写入交易记录

## 6.4 喂食

- 校验背包库存
- 消耗食物数量
- 增加成长值与饱腹度
- 判断是否升级
- 写入成长事件或交易记录

## 6.5 升级

- 根据成长值阈值更新等级
- 升级后触发弹窗或动画
- 可奖励免费积分或食物

## 7. 页面开发顺序

1. 数据模型与本地存储
2. 认领页
3. 动物主页
4. 学习页
5. 商店页
6. 背包页
7. 升级弹窗
8. 记录页

## 8. 开发任务拆解

## Task A：基础框架

- 初始化前端项目
- 配置样式系统
- 建立页面路由
- 定义 TypeScript 类型

## Task B：游戏状态

- 建立全局状态仓库
- 实现本地持久化
- 封装积分、成长、升级规则

## Task C：核心页面

- 首页与认领页
- 动物主页
- 学习页
- 商店和背包

## Task D：反馈系统

- 积分飘字
- 购买成功提示
- 喂食反馈
- 升级弹窗

## Task E：记录系统

- 学习历史列表
- 交易流水
- 升级记录摘要

## 9. 验收标准

- 首次进入可完成认领并进入主页
- 学习完成后积分确实增加
- 积分足够时可购买食物
- 背包内食物可被喂食并消耗
- 成长值达到阈值时自动升级
- 刷新页面后数据仍保留

## 10. 技术建议

- 若追求开发速度：`Next.js + Tailwind + Zustand + LocalStorage`
- 若追求后续扩展：增加 `SQLite` 或 `Supabase`
- 若先做展示原型：可先不接后端，只做本地持久化

## 11. 你下一步可以直接做什么

- 方案确认后，直接开始搭前端脚手架
- 先完成 7 个页面的静态版本
- 再接入状态管理和游戏规则
- 最后补动画、提示和视觉润色
